# Хранилище гильдии и система ДКП

Документ описывает, как в gg-hub устроены **хранилище (банк) гильдии** и связанная с ним **система ДКП**: сущности, права, API, пользовательские сценарии и правила изменения баланса.

## Назначение

**Хранилище гильдии** — каталог предметов гильдии с остатками, тирами, выдачами участникам и историей по каждому предмету.

**Система ДКП** (Dragon Kill Points / очки гильдии) — опциональный режим гильдии. При включении у предметов может быть стоимость в ДКП; выдача списывает очки, отмена выдачи возвращает их. Очки также начисляются по событиям из истории рейдов и вручную с карточки участника. Все движения пишутся в журнал; участники гильдии могут открыть общую историю ДКП со страницы хранилища.

ДКП ведётся **на пользователя в гильдии**, а не на отдельного персонажа. Если у одного аккаунта несколько персонажей в одной гильдии, баланс общий; на карточках персонажей показывается один и тот же остаток.

## Включение ДКП

Флаг `guilds.dkp_enabled` задаётся в настройках гильдии (право `redaktirovanie-dannyx-gildii`). Пока ДКП выключено:

- поле стоимости ДКП у предметов в UI не используется для списаний;
- API баланса, журнала и ручной корректировки отвечает, что система отключена;
- синхронизация ДКП по событиям не создаёт записей в журнале.

## Модель данных

### Банк

| Таблица / сущность | Назначение |
|--------------------|------------|
| `guild_bank_item_tiers` | Тиры предметов гильдии: название и цвет. Привязаны к гильдии. |
| `guild_bank_items` | Предметы: название, описание, тир, `dkp_cost`, `quantity` (`null` = бесконечный остаток). |
| `guild_bank_item_grants` | Факт выдачи: получатель (`received_by_character_id`), кто выдал, причина, `granted_at`, снимок списанных ДКП `dkp_charged`. |

У предмета нет отдельного поля цвета: цвет берётся из тира. Удалить тир можно только если к нему не привязаны предметы.

### ДКП

| Таблица / сущность | Назначение |
|--------------------|------------|
| `guild_user_dkp_balances` | Текущий баланс: пара `(guild_id, user_id)`, значение `balance` (допускается отрицательное). |
| `guild_dkp_ledger_entries` | Журнал движений: подписанное `amount`, `occurred_at`, источник `source`, ссылки на событие / выдачу / предмет / персонажа, `actor_user_id`, `reason`, `balance_after`. |

Источники записи журнала (`GuildDkpLedgerSource`):

- `event` — начисление по сохранённому ДКП события истории;
- `manual` — ручная корректировка с ростера;
- `bank_grant` — списание при выдаче предмета;
- `bank_grant_revoke` — возврат при отмене выдачи.

Баланс обновляется в той же транзакции, что и запись в журнале (`RecordGuildDkpLedgerEntryAction`), с блокировкой строки баланса.

### Связь с историей событий

В `event_histories` хранятся `dkp_base_points`; у участников — `dkp_coefficient` и опциональный `dkp_points_override`. При создании, обновлении или удалении записи истории вызывается `SyncEventHistoryDkpLedgerAction`: старые записи журнала с `source = event` для этого события откатываются и удаляются, затем заново начисляются очки участникам с `character_id` по формуле:

1. если задан `dkp_points_override` — берётся он;
2. иначе `round(dkp_base_points * dkp_coefficient)`.

Участники только с `external_name` (без персонажа в гильдии) в баланс не попадают.

## Права доступа

Все эндпоинты банка и ДКП требуют участия в гильдии (`guild.member`).

| Slug права | Операции |
|------------|----------|
| `dobavliat-predmety` | Тиры (создание/удаление), предметы (создание/изменение/пополнение), просмотр UI управления |
| `udaliat-predmety` | Удаление предмета (если нет активных выдач) |
| `peredavat-predmety-polzovateliam` | Выдача и отмена выдачи, ручное начисление/списание ДКП на ростере |

Просмотр каталога, истории выдач по предмету, баланса ДКП и журнала гильдии доступен участникам гильдии при включённом ДКП (для журнала — при `dkp_enabled`).

## API (префикс `/api/v1/guilds/{guild}`)

### Контекст и банк

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/bank/context` | `my_permission_slugs`, `dkp_enabled`, `dkp_ledger_available` |
| GET | `/bank/tiers` | Список тиров |
| POST | `/bank/tiers` | Создать тир |
| DELETE | `/bank/tiers/{tier}` | Удалить тир без предметов |
| GET | `/bank/items` | Список предметов (тир, остаток, число выдач) |
| POST | `/bank/items` | Создать предмет |
| PUT/PATCH | `/bank/items/{item}` | Изменить предмет / остаток |
| DELETE | `/bank/items/{item}` | Удалить предмет без активных выдач |
| GET | `/bank/items/{item}/grants` | История выдач по предмету |
| POST | `/bank/grants` | Выдать предмет |
| DELETE | `/bank/grants/{grant}` | Отменить выдачу |
| GET | `/bank/members/{character}/grants` | Выдачи, полученные персонажем |

Ответы списков отдают только поля, нужные для экрана (list-resources). Полные ресурсы — при создании/изменении сущности.

### ДКП

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/dkp/ledger` | Журнал движений ДКП гильдии |
| GET | `/members/{character}/dkp` | Баланс пользователя-владельца персонажа |
| POST | `/members/{character}/dkp/adjust` | Ручное изменение (`amount`, опционально `reason`) |

### Выдача предмета и ДКП

При `dkp_enabled` и `item.dkp_cost > 0` при выдаче:

1. Определяется `user_id` владельца получающего персонажа.
2. Если после списания баланс уйдёт в минус, без флага `confirm_negative_balance` API возвращает **422** с полями `data.requires_confirmation`, `balance`, `charged`, `balance_after` и текстом в `errors.confirm_negative_balance`.
3. Повторный запрос с `confirm_negative_balance: true` выполняет выдачу, пишет в журнал `bank_grant` с отрицательным `amount`, сохраняет на гранте `dkp_charged`.
4. Если стоимость 0 или ДКП выключено, списание не выполняется.

При отмене выдачи при ненулевом `dkp_charged` создаётся запись `bank_grant_revoke` с положительным `amount`, затем грант удаляется; остаток предмета на складе увеличивается, как и без ДКП.

Остаток предмета (`quantity`): при выдаче уменьшается на 1 (если не `null`); при отмене — увеличивается на 1. Выдача при нулевом остатке запрещена.

## Пользовательские сценарии (frontend)

Маршруты:

- `/guilds/:id/bank` — хранилище (`GuildBankPage`, FSD: `features/guild-bank`, `widgets/guild-bank`);
- `/guilds/:id/bank/dkp-history` — журнал ДКП;
- `/guilds/:id/roster/:characterId` — карточка участника: баланс ДКП и ручная корректировка (при правах), список полученных предметов.

На странице хранилища: каталог предметов (тир, остаток, выдано, стоимость ДКП в футере карточки), выбор предмета и история выдач, модалки тиров, создания/редактирования, выдачи и отмены. Ссылка «История ДКП» видна при `dkp_ledger_available`. При выдаче в минус показывается подтверждение с текущим балансом и итогом после списания.

Тиры подгружаются при открытии модалки тиров или формы предмета, а не при первой загрузке страницы.

На ростере: блок «ДКП пользователя в гильдии» при включённой системе; форма изменения (обязательное ненулевое целое `amount`, комментарий по желанию) — при `peredavat-predmety-polzovateliam`.

На странице события истории (`events-show`) сохранение блока ДКП приводит к синхронизации журнала на backend (см. выше).

Клиентские API: `frontend/src/shared/api/guildBankApi.ts`, `guildDkpApi.ts`.

## Backend (DDD)

- Домен банка: `backend/Domains/GuildBank/` — модели, actions (списки, CRUD, выдача/отмена), контроллер `GuildBankController`, form requests, JSON resources.
- Домен ДКП: `backend/Domains/GuildDkp/` — баланс, журнал, списание/возврат при выдаче, синхронизация с событиями, `GuildDkpController`.
- Интеграция: `CreateGuildBankItemGrantAction` / `RevokeGuildBankItemGrantAction` вызывают ДКП-actions; `CreateEventHistoryAction`, `UpdateEventHistoryAction`, `DeleteEventHistoryAction` — `SyncEventHistoryDkpLedgerAction`.

Контроллеры не содержат бизнес-логики; валидация в request-классах с понятными сообщениями на русском.

## Ограничения и краевые случаи

- Удаление предмета с активными выдачами запрещено до отмены всех выдач.
- Удаление тира с привязанными предметами запрещено.
- Пересчёт ДКП по старым событиям без повторного сохранения истории в объём документа не входит (нет фоновой миграции балансов).
- Отдельного права только на просмотр журнала ДКП нет: при включённом ДКП журнал доступен участникам гильдии.
