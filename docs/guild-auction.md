# Аукцион гильдии

Документ описывает работу аукциона гильдии в gg-hub: включение, права, пользовательские сценарии, API, realtime-обновления, автоматическое закрытие лотов, выдачу предметов, списание ДКП и уведомления.

## Назначение

Аукцион позволяет выставлять предметы из хранилища гильдии и разыгрывать их между участниками за очки ДКП. Ставки делают пользователи гильдии от имени своих персонажей. После закрытия лота победитель автоматически получает предмет, а с его пользовательского ДКП-баланса списывается сумма выигравшей ставки.

Аукцион доступен только в гильдиях, где включена система ДКП (`guilds.dkp_enabled = true`). Если ДКП выключено, страница и API аукциона возвращают недоступность.

## Основные правила

- Лот создается из предмета хранилища гильдии (`guild_bank_items`).
- На аукцион можно выставить несколько лотов одновременно.
- Начальная стоимость задается при выставлении. Если стоимость не указана, берется стоимость предмета из банка (`guild_bank_items.dkp_cost`), либо `0`.
- У предмета с ограниченным количеством нельзя выставить активных лотов больше, чем доступно единиц на складе.
- Ставка должна быть не ниже начальной стоимости и выше текущей ставки минимум на `1` ДКП.
- Пользователь может ставить только в пределах своего свободного ДКП: текущий баланс минус суммы его лидирующих ставок в других активных лотах.
- Если у пользователя несколько персонажей в этой гильдии, при ставке нужно выбрать персонажа.
- Если ставка сделана менее чем за минуту до окончания лота, окончание продлевается на `10` минут.
- История ставок показывается для выбранного лота.
- Страница отдельного лота доступна любому участнику гильдии.

## Права

| Право | Назначение |
|---|---|
| `dobavliat-predmety-na-aukcion` | Выставлять предметы из хранилища на аукцион |
| `zakryvat-aukcion` | Закрывать лоты вручную |

Обычные участники гильдии могут просматривать аукцион и делать ставки, если ДКП включено и у них есть персонаж в гильдии. Отдельного права на просмотр аукциона нет.

## Пользовательские страницы

| URL | Назначение |
|---|---|
| `/guilds/:id/auction` | Список лотов, статистика, история ставок выбранного лота, создание лотов, ставки, ручное закрытие |
| `/guilds/:id/auction/lots/:lotId` | Полная карточка лота: статус, предмет, цены, лидер/победитель, выдача, кто создал/закрыл, история ставок |

В боковом меню пункт «Аукцион» показывается только для гильдий с включенным ДКП.

## Жизненный цикл лота

### 1. Создание

Пользователь с правом `dobavliat-predmety-na-aukcion` открывает страницу аукциона и добавляет один или несколько лотов:

1. выбирает время окончания;
2. выбирает предметы из хранилища;
3. при необходимости задает начальную стоимость;
4. сохраняет лоты.

После создания:

- лот получает статус `active`;
- участникам гильдии создается in-app уведомление со ссылкой на аукцион;
- если включена Discord-галочка `Выставление лота на аукцион`, в Discord отправляется сообщение со ссылкой на страницу лота;
- через Socket.IO отправляется событие `guild-auction:changed`, чтобы пользователи увидели новый лот без перезагрузки.

### 2. Ставки

Участник гильдии делает ставку на активный лот. Backend проверяет:

- лот принадлежит этой гильдии;
- ДКП в гильдии включено;
- лот активен и время окончания еще не наступило;
- ставка не ниже минимальной;
- выбранный персонаж принадлежит пользователю и состоит в этой гильдии;
- у пользователя достаточно свободных ДКП с учетом других активных лидирующих ставок.

После успешной ставки:

- создается запись в `guild_auction_bids`;
- у лота обновляются `current_bid_amount`, `current_bid_user_id`, `current_bid_character_id`;
- при ставке в последнюю минуту `ends_at` сдвигается на `+10 минут`;
- отправляется realtime-событие `guild-auction:changed`.

### 3. Закрытие

Лот может быть закрыт двумя способами:

- вручную пользователем с правом `zakryvat-aukcion`;
- автоматически, когда наступило `ends_at`.

Автоматическое закрытие выполняет команда:

```bash
php artisan guild-auctions:close-expired
```

Команда запланирована в `routes/console.php` на запуск каждую минуту. Для production/dev окружения должен работать Laravel scheduler:

```bash
php artisan schedule:work
```

Если пользователь открывает список аукциона или страницу лота, истекшие активные лоты этой гильдии также закрываются перед отдачей данных. Это снижает задержку, если scheduler отработал позже.

## Что происходит при закрытии

Если ставок нет:

- лот получает статус `closed`;
- победитель и выдача не создаются;
- при включенной Discord-галочке `Закрытие лота` отправляется сообщение, что победителя нет.

Если есть выигравшая ставка:

1. Проверяется наличие предмета на складе.
2. Создается выдача предмета в `guild_bank_item_grants`.
3. Создается запись журнала ДКП в `guild_dkp_ledger_entries` с отрицательной суммой ставки и source `auction`.
4. Баланс победившего пользователя уменьшается на сумму ставки.
5. Если у предмета задано количество, остаток уменьшается на `1`.
6. У лота заполняются `winner_user_id` и `guild_bank_item_grant_id`.
7. Пользователям с правами `zakryvat-aukcion` или `dobavliat-predmety-na-aukcion` создается in-app уведомление о победителе со ссылкой на лот.
8. Если включена Discord-галочка `Закрытие лота`, отправляется сообщение в Discord с победителем, предметом, ценой и ссылкой на лот.
9. Через Socket.IO отправляется `guild-auction:changed`.

ДКП ведется на пользователя в гильдии, а не на отдельного персонажа. Персонаж в ставке нужен для отображения и для выдачи предмета.

## Таблицы БД

| Поле | Назначение |
|---|---|
| `guild_id` | Гильдия |

| `created_by_user_id` | Кто создал лот |
| `closed_by_user_id` | Кто закрыл вручную; `null` при автоматическом закрытии |

| `guild_bank_item_grant_id` | Связанная выдача предмета |

| `current_bid_user_id` | Пользователь-лидер |
| `current_bid_character_id` | Персонаж, от имени которого сделана лидирующая ставка |
| `status` | `active`, `closed`, `cancelled` |
| `ends_at` | Время окончания |
| `closed_at` | Время закрытия |

## API

Все маршруты требуют авторизации. Все маршруты аукциона также требуют членства в гильдии (`guild.member`). Если ДКП выключено, маршруты аукциона недоступны.

| Метод | Путь | Право | Назначение |
|---|---|---|---|
| GET | `/api/v1/guilds/{guild}/auction/context` | участник | Контекст аукциона: права, ДКП-баланс, персонажи пользователя |
| GET | `/api/v1/guilds/{guild}/auction/lots` | участник | Список лотов гильдии |
| GET | `/api/v1/guilds/{guild}/auction/lots/{lot}` | участник | Карточка одного лота |
| POST | `/api/v1/guilds/{guild}/auction/lots` | `dobavliat-predmety-na-aukcion` | Создать один или несколько лотов |
| POST | `/api/v1/guilds/{guild}/auction/lots/{lot}/bid` | участник | Сделать ставку |
| POST | `/api/v1/guilds/{guild}/auction/lots/{lot}/close` | `zakryvat-aukcion` | Закрыть лот вручную |

### Создание лотов

```json
{
  "ends_at": "2026-05-25T18:00:00.000Z",
  "lots": [
    {
      "guild_bank_item_id": 10,
      "start_price": 20
    },
    {
      "guild_bank_item_id": 11,
      "start_price": null
    }
  ]
}
```

### Ставка

```json
{
  "amount": 25,
  "character_id": 123
}
```

Если у пользователя один персонаж в гильдии, `character_id` может быть не передан. Если персонажей несколько, выбор обязателен.

## Realtime

Аукцион использует Socket.IO через `socket_server`.

Клиент подписывается на комнаты гильдий:

- `guild-auctions:join`
- `guild-auctions:leave`

Backend после изменения лота вызывает `GuildAuctionSocketBroadcaster`, который отправляет запрос в socket server:

- `POST /guild-auctions/broadcast-changed`

Клиенты получают:

- `guild-auction:changed`

Payload:

```json
{
  "guildId": 1,
  "lotId": 20
}
```

После получения события frontend перезагружает данные лотов или конкретного лота.

## Уведомления

### In-app

При выставлении предметов на аукцион уведомление получают участники гильдии. Ссылка ведет на `/guilds/{guild}/auction`.

При закрытии лота с победителем уведомление получают пользователи с правами:

- `zakryvat-aukcion`;
- `dobavliat-predmety-na-aukcion`.

Ссылка ведет на `/guilds/{guild}/auction/lots/{lot}`.

### Discord

Discord-уведомления работают через `SendGuildDiscordNotificationAction` и `notification-gg-hub`. В настройках гильдии на вкладке Discord должны быть:

1. заполнен URL webhook;
2. включена нужная галочка.

Поддерживаемые галочки аукциона:

- `Выставление лота на аукцион`;
- `Закрытие лота`.

Ссылки строятся через `GuildLinkBuilder::auctionLotUrl()`, поэтому учитывают игровой subdomain.

## Ограничения и важные замечания

- Лот нельзя создать, если все доступные экземпляры предмета уже выставлены в активных лотах.
- Если предмет закончился на складе до закрытия лота, закрытие с выдачей вернет ошибку.
- Автозакрытие зависит от работы Laravel scheduler.
- ДКП-баланс проверяется при ставке. После победы списание выполняется при закрытии лота.
- Отмена закрытого лота и возврат предмета/ДКП отдельным сценарием не реализованы.
- Статус `cancelled` предусмотрен в модели, но отдельный пользовательский сценарий отмены лота пока не описан.
